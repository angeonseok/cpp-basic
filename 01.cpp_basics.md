## 프로그램 구조와 출력

### #include / main / 컴파일·실행

> C++은 **컴파일 언어**다. 소스(`.cpp`)를 기계어로 번역(컴파일)한 뒤 실행한다. 파이썬처럼 작성 후 바로 실행되지 않는다.

```cpp
#include <iostream>   // 입출력 기능 가져오기

int main() {          // 프로그램의 시작점
    std::cout << "Hello, World!" << std::endl;
    return 0;         // 정상 종료
}
```

| 구성 요소 | 의미 |
|-----------|------|
| `#include <iostream>` | 입출력 라이브러리 포함 (전처리) |
| `int main()` | 실행이 시작되는 함수 (필수) |
| `std::cout` | 표준 출력 |
| `;` | 모든 문장의 끝 (필수) |
| `return 0;` | main 종료, `0` = 정상 |

컴파일 & 실행 (터미널):
```bash
g++ hello.cpp -o hello   # 컴파일 → 실행파일 hello 생성
./hello                  # 실행
```

> 💡 파이썬은 `python hello.py`로 바로 돌지만, C++은 **컴파일 → 실행** 2단계다. 문장 끝 `;`이나 `main()`이 없으면 컴파일 단계에서 에러가 난다.

### 변수와 자료형

> C++은 **정적 타입** 언어다. 변수를 쓰기 전에 **타입을 명시해 선언**해야 한다. (파이썬은 동적 타입이라 선언이 필요 없었다)

```cpp
int age = 25;               // 정수
double pi = 3.14;           // 실수
char grade = 'A';           // 문자 1개 (작은따옴표!)
bool isOk = true;           // 참/거짓
std::string name = "건석";  // 문자열 (#include <string>)
```

주요 자료형:

| 타입 | 용도 | 크기 | 범위(대략) |
|------|------|------|------------|
| `int` | 정수 | 4 byte | 약 ±21억 |
| `long long` | 큰 정수 | 8 byte | 약 ±9.2×10¹⁸ |
| `double` | 실수 | 8 byte | 유효숫자 ~15자리 |
| `char` | 문자 1개 | 1 byte | -128 ~ 127 |
| `bool` | 참/거짓 | 1 byte | `true` / `false` |
| `std::string` | 문자열 | 가변 | — |

> ⚠️ **가장 흔한 함정 — 정수 오버플로.** 파이썬 `int`는 무한히 커지지만, C++ `int`는 약 21억에서 넘쳐 **값이 망가진다.** 합·곱으로 큰 수가 나오는 문제는 `long long`을 써라.
> ```cpp
> int a = 2000000000;
> int b = a + a;                   // ❌ 오버플로 → 쓰레기 값(음수 등)
> long long c = (long long)a + a;  // ✅ 4000000000 정상
> ```

선언 방식:
```cpp
int x = 5;       // 복사 초기화
int y{5};        // 중괄호 초기화 (좁힘 변환 방지, 권장)
int z;           // 초기화 안 함 → ⚠️ 쓰레기 값 (파이썬엔 없던 상태)
auto w = 5;      // 타입 자동 추론 → int
```

### cout (출력)

> `std::cout`에 `<<`로 값을 흘려보낸다. `<<`는 여러 개를 **연달아** 이어 쓸 수 있다.

```cpp
std::cout << "이름: " << name << ", 나이: " << age << "\n";
// 이름: 건석, 나이: 25

std::cout << 1 + 2 << std::endl;   // 3 (식도 바로 출력 가능)
```

| 줄바꿈 | 차이 |
|--------|------|
| `"\n"` | 단순 줄바꿈 (빠름) |
| `std::endl` | 줄바꿈 + 버퍼 비우기(flush) → 느림 |

> 💡 출력이 많은 문제에선 `std::endl` 대신 `"\n"`이 빠르다. (`endl`은 매번 flush해서 느려진다)

## 입출력

### cin (입력)

> `std::cin`에 `>>`로 입력을 받는다. 변수 타입에 맞춰 **알아서 변환**해준다. (파이썬처럼 `int(input())` 할 필요 없음)

```cpp
int n;
std::cin >> n;          // 정수 입력 → n에 저장

int a, b;
std::cin >> a >> b;     // 공백/줄바꿈으로 구분된 두 값
std::cout << a + b << "\n";
```

> ⚠️ `cin >>`은 **공백·줄바꿈을 기준으로 끊는다.** 문자열에 공백이 있으면 첫 단어만 읽힌다.
> ```cpp
> std::string s;
> std::cin >> s;        // "hello world" 입력 → s = "hello" (world 누락)
> ```

공백 포함 한 줄 전체를 받으려면 `getline`:
```cpp
std::string line;
std::getline(std::cin, line);   // 줄 전체 → "hello world"
```

> 💡 **코딩테스트 입출력 가속.** `main` 첫 줄에 넣으면 cin/cout이 훨씬 빨라진다.
> ```cpp
> ios_base::sync_with_stdio(false);
> cin.tie(NULL);
> ```

### 입력 형변환

> 파이썬은 `input()`이 항상 문자열이라 `int()`로 바꿔야 했지만, C++ `cin`은 변수 타입대로 바로 받는다. 문자열↔숫자 변환이 필요할 때만 함수를 쓴다.

```cpp
std::string s = "42";
int n = std::stoi(s);                  // string → int    → 42
double d = std::stod("3.14");          // string → double → 3.14
std::string t = std::to_string(100);   // 숫자   → string → "100"
```

명시적 형변환(캐스팅):
```cpp
int a = 7, b = 2;
double r  = (double)a / b;                 // C 스타일      → 3.5
double r2 = static_cast<double>(a) / b;    // C++ 권장 방식 → 3.5
```

암시적 형변환(자동):
```cpp
double x = 3 + 1.5;    // int 3 → double → 4.5
int y = 3.9;           // double → int → 3 (⚠️ 반올림 아님, 소수부 버림)
```

## 연산자

### 산술 / 비교 / 논리 / 복합

산술 연산자:

| 연산자 | 의미 | 예시 | 결과 |
|--------|------|------|------|
| `+` `-` `*` | 덧·뺄·곱 | `3 * 2` | `6` |
| `/` | 나눗셈 | `7 / 2` | `3` ⚠️ |
| `%` | 나머지 | `7 % 2` | `1` |
| `++` `--` | 증가·감소 | `x++` | `x = x + 1` |

> ⚠️ **정수끼리 `/`는 몫만 남기고 소수부를 버린다(정수 나눗셈).** 파이썬과 다르다.
> ```cpp
> 7 / 2     // 3    (파이썬은 3.5)
> 7.0 / 2   // 3.5  (한쪽이 실수면 실수 나눗셈)
> 7 % 2     // 1    (나머지)
> ```
> 파이썬 `//`에 해당하는 게 C++의 정수 `/`라고 보면 된다.

비교 연산자 `==` `!=` `>` `<` `>=` `<=` — 결과는 `bool`. 파이썬과 동일.

논리 연산자:

| 연산자 | 의미 | 파이썬 대응 | 예시 |
|--------|------|-------------|------|
| `&&` | 그리고 | `and` | `a > 0 && b > 0` |
| `\|\|` | 또는 | `or` | `a > 0 \|\| b > 0` |
| `!` | 부정 | `not` | `!flag` |

> ⚠️ C++ 논리 연산자는 `&&` `\|\|` `!`다. 파이썬의 `and`/`or`/`not` 키워드 습관이 남으면 헷갈리니 주의. (앞이 결과를 결정하면 뒤를 평가 안 하는 **단축 평가**는 파이썬과 동일)

복합 대입 연산자 `+=` `-=` `*=` `/=` `%=` — 파이썬과 동일.

### 연산자 우선순위

| 우선순위 | 연산자 |
|----------|--------|
| 높음 | `!`, `++`, `--`, 단항 `-` |
| | `*`, `/`, `%` |
| | `+`, `-` |
| | `<`, `<=`, `>`, `>=` |
| | `==`, `!=` |
| | `&&` |
| 낮음 | `\|\|` |

> 💡 헷갈리면 괄호로 묶어라. 가독성과 정확성을 둘 다 잡는다.

## 참고

### 네임스페이스 std

> C++ 표준 라이브러리는 `std`라는 네임스페이스 안에 들어 있다. 그래서 `std::cout`처럼 `std::`를 붙인다.

```cpp
#include <iostream>
using namespace std;   // 이 선언 후엔 std:: 생략 가능

int main() {
    cout << "Hello\n";  // std::cout → cout
}
```

| 방식 | 장점 | 단점 |
|------|------|------|
| `std::` 매번 붙이기 | 출처가 명확 | 길어짐 |
| `using namespace std;` | 짧고 편함 | 이름 충돌 위험 |

> 💡 코딩테스트용 단일 파일에선 `using namespace std;`가 편하다. 큰 프로젝트에선 충돌 방지를 위해 `std::`를 명시하는 걸 권장.

### sizeof와 자료형 크기·범위

> `sizeof`는 자료형/변수가 차지하는 **바이트 수**를 알려준다. 오버플로를 피하려면 타입의 범위를 알아야 한다.

```cpp
std::cout << sizeof(int) << "\n";        // 4
std::cout << sizeof(long long) << "\n";  // 8
std::cout << sizeof(double) << "\n";     // 8
```

| 타입 | 크기 | 최댓값(대략) | 언제 쓰나 |
|------|------|--------------|-----------|
| `int` | 4 byte | 약 +21억 | 기본 정수 |
| `long long` | 8 byte | 약 +9.2×10¹⁸ | 큰 수 / 합·곱 누적 |
| `double` | 8 byte | 매우 큼 | 실수 |

> ⚠️ "최대 10⁵개의 수, 각 10⁹ 이하"의 **합**을 구하는 문제라면 결과가 10¹⁴까지 갈 수 있다 → `int`(±21억) 초과! 이럴 땐 `long long`을 써야 한다. 코딩테스트 단골 실수다.
