# 🔷 C++ 기초 문법 — 객체 지향 (OOP)

> 파이썬에서 클래스를 다뤄봤다면 개념은 이어진다. 다만 C++은 **생성자/소멸자, 접근 지정자, virtual**을 명시적으로 다룬다. (검증·UVM의 SystemVerilog OOP로 거의 그대로 전이됨)

---

## 클래스 기초

### 접근 지정자

> 멤버의 공개 범위를 정한다. 파이썬은 `_` 관례뿐이었지만, C++은 **문법으로 강제**한다.

```cpp
class Dog {
private:               // 외부 접근 불가
    int age;
public:                // 외부 접근 가능
    std::string name;
    void bark() { std::cout << "멍멍\n"; }
};
```

| 지정자 | 접근 범위 | 파이썬 대응 |
|--------|-----------|-------------|
| `public` | 어디서나 | (기본) |
| `private` | 클래스 내부만 | `__name` 관례 |
| `protected` | 클래스 + 자식 | `_name` 관례 |

> 💡 `class`는 기본이 `private`, `struct`는 기본이 `public`. 나머지는 동일하다.

### 생성자 / 소멸자

> **생성자** : 객체 생성 시 자동 호출 (파이썬 `__init__`). **소멸자** : 객체 소멸 시 자동 호출 (파이썬엔 거의 없던 개념, RAII의 핵심).

```cpp
class Dog {
    std::string name;
public:
    Dog(std::string n) : name(n) {     // 생성자 (멤버 초기화 리스트)
        std::cout << name << " 태어남\n";
    }
    ~Dog() {                            // 소멸자 (~클래스명)
        std::cout << name << " 소멸\n";
    }
};

Dog d("바둑이");   // "바둑이 태어남"
                   // 범위 끝나면 "바둑이 소멸" 자동 호출
```

> 💡 `: name(n)`은 **멤버 초기화 리스트**다. 생성자 본문에서 `name = n` 하는 것보다 효율적이라 권장.

### this 포인터

> 멤버 함수 안에서 **자기 자신(객체)을 가리키는 포인터**. 파이썬 `self`에 대응하지만, C++은 자동이라 보통 생략한다.

```cpp
class Dog {
    std::string name;
public:
    void setName(std::string name) {
        this->name = name;   // this->name(멤버) = name(매개변수)
    }
};
```

## 멤버

### static / const 멤버

> **static 멤버** : 모든 객체가 공유 (파이썬 클래스 변수). **const 멤버 함수** : 객체 상태를 바꾸지 않음을 보장.

```cpp
class Counter {
public:
    static int count;          // 공유 변수
    int value = 0;

    int getValue() const {     // const 함수: 멤버 수정 안 함
        return value;
    }
};
int Counter::count = 0;        // static 멤버는 외부에서 정의
```

### 연산자 오버로딩

> `+`, `==` 같은 연산자를 클래스에 맞게 정의. (파이썬의 `__add__` 등 매직 메서드에 대응)

```cpp
class Point {
public:
    int x, y;
    Point(int x, int y) : x(x), y(y) {}

    Point operator+(const Point& other) const {   // + 정의
        return Point(x + other.x, y + other.y);
    }
};

Point a(1, 2), b(3, 4);
Point c = a + b;   // operator+ 호출 → (4, 6)
```

## 상속

> 부모 클래스의 멤버를 물려받는다. `: public 부모`로 표기. (파이썬 `class Dog(Animal)`)

```cpp
class Animal {
public:
    void eat() { std::cout << "먹는다\n"; }
};

class Dog : public Animal {    // Animal 상속
public:
    void bark() { std::cout << "멍멍\n"; }
};

Dog d;
d.eat();   // 부모에게서 물려받음
d.bark();  // 자식 고유
```

### 접근 지정자와 상속

| 상속 방식 | 효과 |
|-----------|------|
| `: public Base` | 부모 public→public 유지 (일반적) |
| `: protected Base` | 부모 public→protected |
| `: private Base` | 부모 멤버 전부 private화 |

> 💡 대부분 `public` 상속을 쓴다. (SystemVerilog의 `extends`가 public 상속에 해당)

### super 호출

> 자식 생성자에서 부모 생성자를 호출하려면 **초기화 리스트**에 적는다. (파이썬 `super().__init__()`)

```cpp
class Dog : public Animal {
    std::string name;
public:
    Dog(std::string n) : Animal(), name(n) {   // 부모 생성자 명시 호출
        // 부모에 인자 필요하면 : Animal(n), name(n)
    }
};
```

## 다형성

### virtual / 오버라이딩

> 부모 메서드를 `virtual`로 두면, 자식이 재정의(오버라이딩)했을 때 **실제 객체 타입의 메서드가 호출**된다. (파이썬은 기본이 이 동작)

```cpp
class Animal {
public:
    virtual void speak() { std::cout << "...\n"; }   // virtual
};

class Dog : public Animal {
public:
    void speak() override { std::cout << "멍멍\n"; }  // override
};

Animal* a = new Dog();
a->speak();   // "멍멍"  (포인터는 Animal이지만 실제는 Dog → 다형성)
delete a;
```

> ⚠️ `virtual`을 안 붙이면 **포인터 타입 기준**으로 호출돼 `"..."`이 나온다. 다형성을 원하면 `virtual` 필수. (파이썬엔 없던 명시 단계)

### 추상 클래스 / 순수 가상 함수

> `= 0`으로 선언한 **순수 가상 함수**가 있으면 그 클래스는 **추상 클래스**(객체 생성 불가, 자식이 반드시 구현). (파이썬 ABC에 대응)

```cpp
class Shape {
public:
    virtual double area() = 0;   // 순수 가상 함수
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() override { return 3.14 * r * r; }  // 반드시 구현
};

// Shape s;       ❌ 추상 클래스라 생성 불가
Circle c(5);      // ✅
```

### 가상 소멸자

> 부모 포인터로 자식 객체를 `delete`할 때, 소멸자가 `virtual`이 아니면 **자식 소멸자가 안 불려 누수**가 난다.

```cpp
class Animal {
public:
    virtual ~Animal() {}   // ✅ 가상 소멸자 (다형성 쓰면 필수)
};
```

> ⚠️ 상속·다형성을 쓸 클래스의 소멸자는 **반드시 `virtual`**로.

## 참고

### 복사 생성자

> 객체를 **복사**할 때 호출되는 생성자. 포인터 멤버가 있으면 직접 정의해야 할 때가 있다(깊은 복사).

```cpp
class Box {
public:
    int value;
    Box(const Box& other) {    // 복사 생성자
        value = other.value;
    }
};
```

### friend

> 특정 함수/클래스에게 `private` 멤버 접근을 허용. (캡슐화를 깨므로 신중히)

```cpp
class Box {
    int secret = 42;
    friend void peek(const Box&);   // peek에게 접근 허용
};
void peek(const Box& b) { std::cout << b.secret; }
```

### SystemVerilog로의 다리

> 이 파일의 OOP 골격이 **검증(SystemVerilog/UVM)으로 거의 그대로 전이**된다.

| C++ | SystemVerilog |
|-----|----------------|
| `class A : public B` | `class A extends B` |
| 생성자 `A(...)` | `function new(...)` |
| 부모 호출 (초기화 리스트) | `super.new(...)` |
| `virtual void f()` | `virtual function void f()` |
| 오버라이딩 | 재정의 |

> 💡 단 SystemVerilog는 포인터·`new`/`delete` 대신 **핸들**을 쓰고 메모리를 자동 관리한다. 개념은 같고 메모리 모델만 다르다.
