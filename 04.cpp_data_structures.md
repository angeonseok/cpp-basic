# 🔷 C++ 기초 문법 — 자료구조 (배열 · 문자열 · STL)

> C++의 자료구조는 대부분 **표준 라이브러리(STL)**로 제공된다. 파이썬의 내장 list·dict·set에 대응하는 것들을 여기서 익힌다.

---

## 배열

### 1차원 배열

> 같은 타입 값을 **고정 개수**로 연속 저장. C 스타일 배열은 **크기가 고정**이다. (파이썬 list는 가변 — 큰 차이)

```cpp
int arr[5] = {1, 2, 3, 4, 5};
arr[0] = 10;                 // 인덱싱 (0부터)
std::cout << arr[0];         // 10

int zeros[5] = {};           // 전부 0으로 초기화
```

> ⚠️ C 배열은 **크기 변경 불가, 경계 검사 없음.** `arr[10]`처럼 범위를 넘어도 컴파일은 되지만 **쓰레기 값/충돌**이 난다. 가변 크기가 필요하면 아래 `vector`를 써라.

### 2차원 배열

```cpp
int grid[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};
std::cout << grid[1][2];   // 6  (행 1, 열 2)
```

## 문자열

### char 배열 vs std::string

> C 스타일 `char[]`도 있지만, 실무·코테에선 **`std::string`**을 쓴다. 가변 길이에 편리한 함수가 많고, 파이썬 str에 가장 가깝다.

```cpp
#include <string>
std::string s = "hello";
s += " world";              // 이어붙이기 → "hello world"
std::cout << s[0];          // 'h' (인덱싱)
std::cout << s.length();    // 11 (길이)
```

### 주요 문자열 함수

| 함수 | 설명 | 예시 → 결과 |
|------|------|-------------|
| `.size()` / `.length()` | 길이 | `s.size()` |
| `.substr(i, n)` | 부분 문자열 | `s.substr(0, 3)` → `"hel"` |
| `.find("x")` | 위치 (없으면 `string::npos`) | `s.find("o")` → `4` |
| `+` | 이어붙이기 | `"a" + b` |
| `std::stoi(s)` | 문자열 → 정수 | `stoi("42")` → `42` |

> ⚠️ 파이썬 문자열은 **불변**이라 수정 시 새 객체였지만, C++ `std::string`은 **가변**이라 `s[0] = 'H'`로 직접 수정된다.

## STL 컨테이너

### vector — 가변 배열 (≈ 파이썬 list)

> 크기가 자동으로 늘어나는 배열. **C++에서 가장 많이 쓰는 컨테이너.** 파이썬 list라고 보면 된다.

```cpp
#include <vector>
std::vector<int> v = {1, 2, 3};
v.push_back(4);          // 끝에 추가 → {1,2,3,4}
v[0] = 10;               // 인덱싱
std::cout << v.size();   // 4
v.pop_back();            // 끝 제거

std::vector<int> v2(5, 0);   // 0이 5개인 벡터
std::vector<std::vector<int>> grid(3, std::vector<int>(4, 0)); // 3x4 2차원
```

### map / unordered_map — 키-값 (≈ 파이썬 dict)

> 키로 값을 저장. `map`은 **키 정렬**(트리), `unordered_map`은 **해시**(빠름, 순서 없음).

```cpp
#include <map>
std::map<std::string, int> age;
age["건석"] = 25;             // 추가/수정
std::cout << age["건석"];     // 25
if (age.count("철수")) { }    // 키 존재 확인

for (auto& p : age) {                // 순회 (p.first=키, p.second=값)
    std::cout << p.first << ":" << p.second << "\n";
}
```

| | `map` | `unordered_map` |
|--|-------|------------------|
| 내부 | 트리(정렬됨) | 해시 |
| 순서 | 키 오름차순 | 없음 |
| 속도 | O(log n) | 평균 O(1) |

> 💡 빠른 조회만 필요하면 `unordered_map`, 키 정렬이 필요하면 `map`. (파이썬 dict는 `unordered_map`에 가깝다)
> ⚠️ C++17부터는 `for (auto& [key, val] : age)` (구조적 바인딩)로 더 짧게 쓸 수 있지만, **C++14에선 안 되므로** 위처럼 `p.first`/`p.second`로 접근한다.

### set — 중복 없는 집합 (≈ 파이썬 set)

```cpp
#include <set>
std::set<int> s = {3, 1, 2, 1};   // 중복 제거 → {1, 2, 3} (정렬됨)
s.insert(5);
if (s.count(2)) { }               // 존재 확인
s.erase(1);                        // 삭제
```

### pair — 두 값 묶기

```cpp
std::pair<int, std::string> p = {1, "건석"};
std::cout << p.first << " " << p.second;   // 1 건석
```

## STL 어댑터

> 특정 동작에 특화된 컨테이너. 알고리즘 트랙(스택·큐 자료구조)과 직결.

| 컨테이너 | 특징 | 주요 연산 |
|----------|------|-----------|
| `stack` | 후입선출(LIFO) | `push` / `top` / `pop` |
| `queue` | 선입선출(FIFO) | `push` / `front` / `pop` |
| `priority_queue` | 최댓값 우선 | `push` / `top` / `pop` |
| `deque` | 양쪽 삽입/삭제 | `push_front` / `push_back` |

```cpp
#include <stack>
#include <queue>
std::stack<int> st;
st.push(1); st.push(2);
std::cout << st.top();   // 2 (마지막에 넣은 것)
st.pop();

std::queue<int> q;
q.push(1); q.push(2);
std::cout << q.front();  // 1 (먼저 넣은 것)
```

## 반복자와 알고리즘

### 반복자 (iterator)

> 컨테이너의 원소를 가리키는 포인터 같은 것. 대부분 **범위 기반 for**로 대체된다.

```cpp
std::vector<int> v = {1, 2, 3};
for (int x : v) {            // 범위 기반 for (≈ 파이썬 for x in v)
    std::cout << x << " ";   // 1 2 3
}
```

### sort / find (`<algorithm>`)

> STL이 제공하는 알고리즘 함수. **반복자 구간**(`begin`, `end`)을 받는다.

```cpp
#include <algorithm>
std::vector<int> v = {3, 1, 2};
std::sort(v.begin(), v.end());     // 오름차순 → {1, 2, 3}

// 람다로 정렬 기준 지정 (내림차순)
std::sort(v.begin(), v.end(), [](int a, int b) {
    return a > b;
});

auto it = std::find(v.begin(), v.end(), 2);  // 2 찾기
if (it != v.end()) { /* 찾음 */ }
```

## 참고

### 범위 기반 for + auto

> 타입을 `auto`로 추론하면 순회가 간결해진다. 원본을 수정하려면 `auto&`.

```cpp
std::vector<int> v = {1, 2, 3};
for (auto x : v) { }       // 복사본
for (auto& x : v) x *= 2;  // 참조 → 원본 수정 {2,4,6}
```

### emplace vs push

> `push_back`은 **만들어진 객체를 복사/이동**해 넣고, `emplace_back`은 **컨테이너 안에서 바로 생성**해 약간 효율적이다.

```cpp
std::vector<std::pair<int,int>> v;
v.push_back({1, 2});      // pair를 만들어 넣음
v.emplace_back(1, 2);     // 안에서 바로 생성 (인자만 전달)
```

> 💡 입문 단계에선 `push_back`으로 충분하다. `emplace`는 "더 효율적인 버전이 있다" 정도로 알아두면 된다.
