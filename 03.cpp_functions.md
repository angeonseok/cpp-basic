## 함수

### 함수 구조

> 특정 작업을 묶어 이름 붙인 코드. C++은 **반환 타입을 명시**해야 한다. (파이썬은 `def`만 쓰고 타입 생략)

```cpp
int add(int a, int b) {   // 반환타입 함수이름(매개변수)
    return a + b;
}

int main() {
    int s = add(3, 5);    // 8
}
```

| 구성 | 의미 |
|------|------|
| 반환 타입 | 돌려줄 값의 타입 (`int`, `double`...) |
| `void` | 반환값 없음 |
| 매개변수 | 받을 입력 (타입 명시 필수) |
| `return` | 결과 반환 + 함수 즉시 종료 |

```cpp
void greet(std::string name) {   // void = 반환값 없음
    std::cout << "Hi, " << name << "\n";
}
```

### 선언과 정의 분리 (프로토타입)

> C++은 함수를 **호출하기 전에 알고 있어야** 한다. 정의가 아래 있으면 위에 **선언(프로토타입)**만 먼저 적는다. (파이썬엔 없는 개념)

```cpp
int add(int a, int b);   // 선언(프로토타입) — 본문 없이 세미콜론

int main() {
    std::cout << add(3, 5);   // 정의가 아래 있어도 OK
}

int add(int a, int b) {  // 정의
    return a + b;
}
```

> 💡 프로토타입을 헤더 파일(`.h`)에 모아 여러 파일에서 공유하는 게 C++의 모듈화 방식이다.

## 매개변수와 인자

### 값 전달 / 참조 전달 / 포인터 전달

> C++은 인자를 넘기는 방식이 **세 가지**고, **원본을 바꿀 수 있느냐**가 갈린다. (파이썬엔 이 구분이 없어 가장 헷갈리는 부분)

**① 값 전달 (복사)** — 기본. 원본 안 바뀜.
```cpp
void f(int x) { x = 100; }   // 복사본만 변경
int a = 5;
f(a);
std::cout << a;   // 5  (원본 그대로)
```

**② 참조 전달 (`&`)** — 원본을 직접 가리킴. 바뀜.
```cpp
void f(int& x) { x = 100; }  // 원본 변경
int a = 5;
f(a);
std::cout << a;   // 100  (바뀜!)
```

**③ 포인터 전달 (`*`)** — 주소를 넘김. 바뀜.
```cpp
void f(int* x) { *x = 100; }
int a = 5;
f(&a);
std::cout << a;   // 100
```

| 방식 | 문법 | 원본 변경 | 용도 |
|------|------|-----------|------|
| 값 | `int x` | ❌ | 단순 입력 |
| 참조 | `int& x` | ✅ | 원본 수정 / 큰 데이터 복사 회피 |
| 포인터 | `int* x` | ✅ | 주소를 직접 다룰 때 |

> 💡 큰 데이터(벡터 등)는 복사 비용을 피하려 `const T&`(읽기 전용 참조)로 넘기는 게 관례. 자세한 건 `05.cpp_pointer_memory`에서.

### 기본 인자

> 인자를 생략하면 기본값 사용. (파이썬과 동일하나 **뒤쪽 매개변수부터**)

```cpp
int power(int base, int exp = 2) {   // exp 기본값 2
    int r = 1;
    for (int i = 0; i < exp; i++) r *= base;
    return r;
}
power(3);      // 9  (3^2)
power(3, 3);   // 27 (3^3)
```

> ⚠️ 기본값 있는 매개변수는 **반드시 뒤쪽**에. `int f(int a = 1, int b)` ❌

### 함수 오버로딩

> **같은 이름** 함수를 매개변수 구성만 다르게 여러 개 정의. (파이썬엔 없는 기능 — 파이썬은 마지막 정의가 덮어씀)

```cpp
int add(int a, int b)          { return a + b; }
double add(double a, double b) { return a + b; }

add(1, 2);       // int 버전 → 3
add(1.5, 2.5);   // double 버전 → 4.0
```

> 컴파일러가 **인자 타입·개수**를 보고 알맞은 버전을 자동 선택한다.

## 참고

### 인라인 함수

> `inline`을 붙이면 호출을 **호출 지점에 코드 펼치기**로 처리해 호출 비용을 줄인다. (작은 함수에 권장, 강제는 아님)

```cpp
inline int square(int x) { return x * x; }
```

### 람다 표현식

> 이름 없는 함수를 그 자리에서 정의. `[캡처](매개변수) { 본문 }` 형태. (파이썬 `lambda`보다 강력 — `[]`로 바깥 변수 캡처)

```cpp
auto add = [](int a, int b) { return a + b; };
std::cout << add(3, 5);   // 8

int base = 10;
auto addBase = [base](int x) { return x + base; };  // base 캡처
std::cout << addBase(5);  // 15
```

| 캡처 | 의미 |
|------|------|
| `[]` | 캡처 없음 |
| `[base]` | base를 값으로 캡처 |
| `[&base]` | base를 참조로 캡처 |
| `[=]` / `[&]` | 전부 값 / 전부 참조 |

> 💡 `sort`의 비교 기준 등 일회성 함수에 주로 쓴다. (`04.cpp_data_structures`의 sort와 연결)

### 재귀 함수

> 자기 자신을 호출. **종료 조건(기저 조건)** 필수. (개념은 파이썬과 동일)

```cpp
int factorial(int n) {
    if (n <= 1) return 1;          // 기저 조건
    return n * factorial(n - 1);   // 재귀 호출
}
factorial(5);   // 120
```

> ⚠️ 파이썬엔 재귀 한도가 있었지만, C++은 한도가 없어도 **스택 메모리 초과(stack overflow)**로 죽는다. 깊은 재귀는 반복문/메모이제이션으로 바꾸는 걸 고려. (알고리즘 트랙의 DFS·DP와 직결)
