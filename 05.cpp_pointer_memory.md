# 🔷 C++ 기초 문법 — 포인터 · 참조 · 메모리

> 파이썬은 메모리 주소·할당을 전부 숨기고 GC가 알아서 관리했다. C++은 **메모리를 직접 다룬다.** 이게 파이썬에서 넘어올 때 가장 큰 개념적 도약이다.

---

## 포인터

### 주소와 역참조

> **포인터** : 다른 변수의 **메모리 주소**를 담는 변수. `&`로 주소를 얻고, `*`로 그 주소의 값에 접근(역참조)한다.

```cpp
int x = 5;
int* p = &x;        // p = x의 주소

std::cout << p;     // 주소값 (예: 0x7ffe...)
std::cout << *p;    // 5   (역참조 → x의 값)

*p = 10;            // 역참조로 값 수정
std::cout << x;     // 10  (x가 바뀜! p가 x를 가리키므로)
```

| 기호 | 의미 | 예시 |
|------|------|------|
| `int* p` | 포인터 선언 | p는 int의 주소를 담음 |
| `&x` | 주소 얻기(address-of) | x의 주소 |
| `*p` | 역참조(dereference) | p가 가리키는 값 |

> 💡 파이썬에도 `id(x)`로 주소 비슷한 걸 봤지만, **역참조해서 값을 바꾸는 일은 없었다.** C++ 포인터는 그 주소를 통해 원본을 직접 조작한다.

### 포인터와 배열

> 배열 이름은 사실상 **첫 원소의 주소**다. 그래서 포인터로 배열을 순회할 수 있다.

```cpp
int arr[3] = {10, 20, 30};
int* p = arr;            // arr == &arr[0]

std::cout << *p;         // 10   (arr[0])
std::cout << *(p + 1);   // 20   (arr[1], 포인터 산술)
std::cout << p[2];       // 30   (포인터도 [] 사용 가능)
```

> ⚠️ 포인터 산술 `p + 1`은 "1바이트 뒤"가 아니라 **"원소 1개 뒤"**다. (int면 4바이트 이동)

### nullptr

> 아무것도 가리키지 않는 포인터. `nullptr`로 초기화한다. (옛 `NULL`보다 `nullptr` 권장)

```cpp
int* p = nullptr;        // 빈 포인터

if (p != nullptr) {      // 가리키는 게 있는지 확인 후 사용
    std::cout << *p;
}
```

> ⚠️ `nullptr`(또는 쓰레기 주소)를 역참조하면 **프로그램이 죽는다**(segfault). 역참조 전에 항상 유효한지 확인하라.

## 참조

### 참조 변수

> **참조(reference)** : 기존 변수의 **별명(alias)**. `&`를 타입에 붙여 선언하며, 한 번 묶이면 그 변수 그 자체처럼 동작한다.

```cpp
int x = 5;
int& ref = x;       // ref는 x의 별명

ref = 10;
std::cout << x;     // 10  (ref를 바꾸면 x가 바뀜)
```

> ⚠️ 참조는 **선언 시 반드시 초기화**해야 하고, 한 번 묶이면 **다른 변수로 바꿀 수 없다**(rebind 불가).

### 포인터 vs 참조

| | 포인터 (`int* p`) | 참조 (`int& r`) |
|--|-------------------|------------------|
| 초기화 | 나중에 가능 | 선언 시 필수 |
| 재지정 | 다른 대상 가리키기 가능 | 불가 (한 대상 고정) |
| null | 가능 (`nullptr`) | 불가 (항상 유효) |
| 문법 | `*p`로 역참조 | 변수처럼 바로 사용 |

> 💡 함수 인자로 원본을 수정할 때, 대부분 **참조(`int&`)가 포인터보다 안전하고 간결**하다. (03번 함수 파일의 "참조 전달"이 이것)

## 메모리

### 스택 vs 힙

> 변수는 두 종류의 메모리 영역에 저장된다.

| 영역 | 특징 | 할당 방식 |
|------|------|-----------|
| 스택(stack) | 빠름, 크기 제한, **자동 해제** | 일반 지역 변수 |
| 힙(heap) | 큼, 느림, **수동 관리** | `new`로 직접 할당 |

```cpp
int a = 5;              // 스택: 함수 끝나면 자동 소멸
int* p = new int(5);    // 힙: 직접 delete 해야 소멸
```

### 동적 할당 (new / delete)

> 실행 중에 힙에서 메모리를 직접 잡고(`new`) 직접 반납(`delete`)한다. (파이썬엔 없던 책임)

```cpp
int* p = new int(42);   // 힙에 int 하나 할당
std::cout << *p;        // 42
delete p;               // 반납 (필수!)

int* arr = new int[5];  // 배열 할당
delete[] arr;           // 배열은 delete[] 로 반납
```

> ⚠️ `new`한 건 반드시 `delete`. 배열은 `delete[]`. 짝을 안 맞추면 문제가 생긴다.

### 메모리 누수

> `delete`를 빠뜨리면 그 메모리는 **반납되지 않고 계속 점유**된다 → 누수(leak). 파이썬은 GC가 알아서 했지만 C++은 직접 챙겨야 한다.

```cpp
void leak() {
    int* p = new int(5);
    // delete p;  ← 빠뜨림 → 누수!
}                  // p는 사라지지만 힙의 메모리는 그대로 남음
```

## 소유권 (Ownership)

### 스마트 포인터

> `new`/`delete`를 손으로 맞추는 건 실수가 잦다. **스마트 포인터**는 범위를 벗어나면 **자동으로 메모리를 반납**한다. (`#include <memory>`)

```cpp
#include <memory>

std::unique_ptr<int> p = std::make_unique<int>(5);  // C++14
std::cout << *p;     // 5
// delete 불필요 — p가 사라질 때 자동 반납
```

| 스마트 포인터 | 의미 |
|---------------|------|
| `unique_ptr` | 소유자 **하나** (복사 불가, 이동만) |
| `shared_ptr` | 여러 소유자 공유 (참조 카운트) |

```cpp
auto sp = std::make_shared<int>(10);  // 공유 소유
auto sp2 = sp;                        // 카운트 2
```

### RAII

> **RAII (Resource Acquisition Is Initialization)** : 자원을 객체가 쥐고, **객체가 사라질 때 소멸자가 자동으로 반납**하는 C++ 핵심 패턴. 스마트 포인터가 바로 이 원리다.

```cpp
{
    auto p = std::make_unique<int>(5);
    // ... 사용 ...
}   // 블록 끝 → p 소멸 → 메모리 자동 반납
```

> 💡 **네가 쓰는 `vector`도 RAII다.** vector는 내부적으로 힙에 메모리를 잡지만, 범위를 벗어나면 알아서 반납한다. 그래서 **현대 C++에선 raw `new`/`delete`를 거의 쓸 일이 없다** — `vector`·스마트 포인터가 대신 관리한다. (포인터 개념은 알되, 실전에선 컨테이너를 쓰는 이유)

## 참고

### const 포인터

> `const` 위치에 따라 "값이 const"인지 "포인터가 const"인지 달라진다.

| 선언 | 의미 |
|------|------|
| `const int* p` | 가리키는 **값** 수정 불가 (포인터 이동은 가능) |
| `int* const p` | **포인터** 재지정 불가 (값 수정은 가능) |
| `const int* const p` | 둘 다 불가 |

```cpp
int x = 5, y = 9;
const int* p = &x;
// *p = 10;   ❌ 값 수정 불가
p = &y;       // ✅ 다른 곳 가리키기는 OK
```

> 💡 함수에 큰 데이터를 **읽기 전용으로** 넘길 때 `const T&`(또는 `const T*`)를 쓴다. 복사 비용 없이 원본 수정도 막는다.

### move 맛보기

> 복사는 데이터를 통째로 베끼지만, **이동(move)**은 소유권만 넘겨 비용을 아낀다. `std::move`로 명시한다.

```cpp
std::vector<int> a = {1, 2, 3};
std::vector<int> b = std::move(a);   // a의 내용을 b로 이동 (복사 아님)
// 이후 a는 비어 있음 (사용하지 않기)
```

> 입문 단계에선 "큰 객체를 넘길 때 복사 대신 이동이 일어난다" 정도만 알아두면 된다. 깊은 내용은 검증·EDA로 갈 때 다시 본다.
