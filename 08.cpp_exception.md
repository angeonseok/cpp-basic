# 🔷 C++ 기초 문법 — 예외 처리

> 실행 중 발생하는 오류를 **프로그램을 죽이지 않고** 처리하는 메커니즘. 개념은 파이썬과 같지만 문법이 다르다 (`try/catch/throw`).

---

## 에러와 예외

### 컴파일 에러 vs 런타임 에러

| 종류 | 시점 | 예시 |
|------|------|------|
| 컴파일 에러 | 빌드 단계 | 세미콜론 누락, 타입 불일치 |
| 런타임 에러 | 실행 중 | 0으로 나눔, 범위 초과 접근 |

> 컴파일 에러는 빌드가 안 되니 바로 안다. **예외 처리는 런타임 에러**를 다룬다.

---

## 예외 처리

### try / catch / throw

> `try` 안에서 문제가 생기면 `throw`로 예외를 던지고, `catch`가 받아 처리한다. (파이썬 `try/except/raise`에 대응)

| C++ | 파이썬 대응 |
|-----|-------------|
| `try` | `try` |
| `catch` | `except` |
| `throw` | `raise` |

```cpp
#include <stdexcept>

try {
    int age = -5;
    if (age < 0) {
        throw std::runtime_error("나이는 음수일 수 없음");   // 예외 던짐
    }
} catch (const std::runtime_error& e) {        // 예외 받음
    std::cout << "에러: " << e.what() << "\n";  // e.what() = 메시지
}

std::cout << "프로그램 계속\n";   // 죽지 않고 실행
```

> 💡 `e.what()`은 예외 메시지를 돌려준다. catch는 보통 **참조(`const ...&`)**로 받는다.

### 복수 예외 처리

> 여러 종류의 예외를 각각 다르게 처리. `...`는 모든 예외를 받는다.

```cpp
try {
    // ...
} catch (const std::out_of_range& e) {   // 구체적인 것 먼저
    std::cout << "범위 초과\n";
} catch (const std::exception& e) {       // 포괄적인 건 뒤에
    std::cout << "일반 예외: " << e.what() << "\n";
} catch (...) {                            // 그 외 전부
    std::cout << "알 수 없는 예외\n";
}
```

> ⚠️ `catch`는 **위에서부터** 검사하므로, 구체적인 예외를 먼저 두고 포괄적인 `std::exception`을 뒤에 둬야 한다. (파이썬 except 순서 원칙과 동일)

## 참고

### std::exception 계층

> 표준 예외들은 `std::exception`을 **상속**한 클래스다. (06번 상속과 연결)

| 예외 | 발생 상황 |
|------|-----------|
| `std::exception` | 모든 표준 예외의 부모 |
| `std::runtime_error` | 실행 중 논리 오류 |
| `std::out_of_range` | 범위 초과 (`vector::at` 등) |
| `std::invalid_argument` | 잘못된 인자 |

### 사용자 정의 예외

> `std::exception`(또는 그 자식)을 상속해 직접 예외 타입을 만든다.

```cpp
#include <stdexcept>

class BalanceError : public std::runtime_error {
public:
    BalanceError(const std::string& msg)
        : std::runtime_error(msg) {}    // 부모 생성자에 메시지 전달
};

try {
    throw BalanceError("잔액 부족");
} catch (const std::exception& e) {     // 부모 타입으로 받기 가능
    std::cout << e.what();   // 잔액 부족
}
```

### RAII와 예외 안전성

> 예외가 던져지면 그 지점 이후 코드는 건너뛴다. 이때 `new`로 잡은 메모리는 **누수**될 수 있다. **RAII(스마트 포인터·컨테이너)**를 쓰면 예외가 나도 소멸자가 자동 호출돼 안전하다.

```cpp
void risky() {
    auto p = std::make_unique<int>(5);   // RAII
    throw std::runtime_error("문제 발생");
}   // 예외로 빠져나가도 p는 자동 반납 (누수 없음)
```

> 💡 raw `new` + 예외 조합은 누수의 단골 원인. 05번에서 본 **스마트 포인터·vector를 쓰면 예외 안전성까지 공짜로 얻는다.** 이게 현대 C++이 RAII를 강조하는 이유다.
