# 🔷 C++ 기초 문법 — 템플릿 · 제네릭

> **템플릿** : 타입을 매개변수처럼 받아 **여러 타입에 동작하는 코드를 한 번만** 작성하는 기능. (파이썬은 동적 타입이라 필요 없던 개념 — C++ 고유)

---

## 템플릿

### 함수 템플릿

> 타입만 다르고 로직이 같은 함수를 **하나로** 작성한다.

```cpp
template <typename T>      // T = 타입 매개변수
T getMax(T a, T b) {
    return (a > b) ? a : b;
}

getMax(3, 7);        // int 버전 → 7
getMax(1.5, 2.5);    // double 버전 → 2.5
getMax('a', 'z');    // char 버전 → 'z'
```

> 💡 `typename` 대신 `class`를 써도 같다 (`template <class T>`). 컴파일러가 호출 시 타입을 보고 **버전을 자동 생성**한다.

### 클래스 템플릿

> 클래스 전체를 타입에 대해 일반화. (STL의 `vector<T>`, `map<K,V>`가 전부 이 방식)

```cpp
template <typename T>
class Box {
    T value;
public:
    Box(T v) : value(v) {}
    T get() { return value; }
};

Box<int> a(5);            // T = int
Box<std::string> b("hi"); // T = string
std::cout << a.get();     // 5
```

> ⚠️ C++14에선 클래스 템플릿 사용 시 **타입을 명시**해야 한다 (`Box<int>`). 타입 생략(`Box a(5)`)은 C++17(CTAD)부터다.

## 제네릭 프로그래밍

### 타입 매개변수

> 타입을 매개변수로 받아 **타입에 독립적인** 알고리즘·자료구조를 만드는 방식. 여러 타입 매개변수도 가능.

```cpp
template <typename K, typename V>
struct Pair {
    K key;
    V value;
};

Pair<std::string, int> p{"age", 25};
```

기본 타입 인자도 줄 수 있다:
```cpp
template <typename T = int>   // 기본 타입 int
class Stack { /* ... */ };

Stack<> s;        // T = int (기본)
Stack<double> d;  // T = double
```

## 참고

### STL과 템플릿의 관계

> 네가 04번에서 쓴 STL이 전부 템플릿으로 만들어졌다.

| 사용 | 실제 정체 |
|------|-----------|
| `std::vector<int>` | 클래스 템플릿 `vector<T>`에 `T = int` |
| `std::map<string,int>` | `map<K,V>`에 `K = string, V = int` |
| `std::sort(...)` | 함수 템플릿 |

> 💡 즉 STL을 쓰는 순간 이미 템플릿을 쓰고 있던 것. 직접 템플릿을 작성하는 건 **자료구조·알고리즘을 라이브러리처럼 재사용**할 때 필요하고, 검증·EDA 코드에서 자주 등장한다.
